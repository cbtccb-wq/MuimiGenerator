# Evaluation Rubric — MuimiGenerator

採点はこのファイルの基準のみを使用する。ループ中に採点軸を書き換えない。

---

## 重み配分

| カテゴリ | 配点 |
|---|---|
| 機能 | 40点 |
| 安全性 | 25点 |
| 可読性 | 15点 |
| 性能 | 10点 |
| 見た目 | 10点 |
| **合計** | **100点** |

---

## 合格判定ルール

- **総合80点以上** → 合格
- **安全性 20/25 未満** → 総合点に関係なく**不合格**
- **機能 35/40 未満** → 改善ループ継続
- **見た目の改善ループは最大2回まで**（機能・安全性を満たした後のみ追う）

---

## 機能（40点）

| 項目 | 配点 | 基準 |
|---|---|---|
| 主要機能が動作する | 20 | クラッシュなし・正常出力 |
| 境界値で正しく動く | 10 | 空・null・0件・超長文で破綻しない |
| 主要操作を20回連続で実行して破綻しない | 10 | 連打・繰り返しに耐える |

## 安全性（25点）

| 項目 | 配点 | 基準 |
|---|---|---|
| APIキー・秘密情報のハードコードなし | 10 | 0件 |
| 内部IDや変数名を画面に表示しない | 8 | 0件 |
| エラーメッセージの生出力なし | 7 | ユーザー向けメッセージに変換されている |

## 可読性（15点）

| 項目 | 配点 | 基準 |
|---|---|---|
| lint error | 8 | 0件 |
| type error | 7 | 0件 |

## 性能（10点）

| 項目 | 配点 | 基準 |
|---|---|---|
| 操作後の画面更新 | 5 | 1秒以内 |
| console error / warning | 5 | 0件 |

## 見た目（10点）

| 項目 | 配点 | 基準 |
|---|---|---|
| フォントサイズ種 | 3 | 3種類以内 |
| 同時表示ボタン数 | 3 | 最大4個 |
| 画面縦長・横長で致命的崩れなし | 4 | レイアウト破綻なし |

---

## 足切り条件

以下が1件でも検出されたら、他のスコアに関係なく即不合格：

- APIキー・秘密情報のハードコード
- main ブランチへの直接 push
- 既存テストの削除
- 破壊禁止領域への無承認変更

---

## テスト合格基準

- unit test: 100% pass
- coverage: 80%以上（目標）
- Level A テスト: 毎ループ全通
